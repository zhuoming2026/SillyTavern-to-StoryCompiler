# 环境检测与兼容方案

> 各技能在启动时通过此流程检测可用工具，选择最佳执行路径。
> 适用于 Reasonix 在 Windows / macOS / Linux 等不同平台上的兼容适配。

---

## 一、检测流程

执行顺序如下，命中的第一条即为当前可用路径：

| 优先级 | 检测项目 | 判断方式 | 适用场景 |
|:------|:---------|:---------|:---------|
| 1️⃣ | **bash 可用** | 执行 `bash -c "echo ok"` 返回成功 | 全功能（macOS / Linux） |
| 2️⃣ | **python 可用** | 执行 `python --version` 返回成功 | Python 脚本执行（Windows / Linux） |
| 3️⃣ | **node 可用** | 执行 `node --version` 返回成功 | Node.js 脚本执行（所有平台） |
| 4️⃣ | **仅文件工具** | 以上均不可用 | 纯读写操作，无脚本执行能力 |

### 检测示例（以 python 为例）

```
尝试: bash -c "python --version"
  ↓ 成功 → 标记 env.python = bash("python")
  ↓ 失败
尝试: python --version
  ↓ 成功 → 标记 env.python = python
  ↓ 失败
尝试: python3 --version
  ↓ 成功 → 标记 env.python = python3
  ↓ 失败
标记 env.python = unavailable
```

## 二、各平台已知情况

| 平台 | bash | python | node | 备注 |
|:-----|:-----|:-------|:-----|:------|
| **macOS** | ✅ 可用 | ✅ python3 | ✅ | 全功能，沙箱支持 |
| **Linux** | ✅ 可用 | ✅ python3 | ✅ | 全功能 |
| **Windows (Git Bash)** | ✅ 可用（无沙箱） | ✅ python | ✅ | 路径用 `/c/...` 风格 |
| **Windows (Reasonix 沙箱拦截)** | ❌ 被拦截 | ❌ 无法执行 | ❌ 无法执行 | 仅文件工具可用 |

> ⚠️ **Windows 沙箱拦截情况**：如果 `reasonix.toml` 中 `bash = "enforce"` 但沙箱不可用（Windows 不支持），所有 bash 命令会被宿主拦截。此时需要：
> - 注释或删除 `bash = "enforce"` 配置行
> - 或改用 `bash = "off"`

## 三、技能中的兼容写法

### 3.1 PNG 解析（唯一需要脚本执行的步骤）

SillyTavern 角色卡以 PNG 格式存储时，JSON 数据嵌入在 `tEXt` chunk 的 `chara` 字段中（base64 编码）。

**全平台兼容方案：**

当检测到 bash/python 不可用时，技能应：

1. **写入 Python 脚本到临时文件**（如 `.reasonix/tmp/extract_chara.py`）
2. **指引用户手动执行**：
   - 生成 `.bat`（Windows）或 `.sh`（macOS/Linux）脚本
   - 告知用户执行命令
3. **用户执行完成后，技能继续后续文件操作**

**Python 脚本内容示例（跨平台兼容）：**

```python
import struct, base64, json, sys, os

with open(sys.argv[1], 'rb') as f:
    data = f.read()

pos = 8
while pos < len(data):
    length = struct.unpack('>I', data[pos:pos+4])[0]
    chunk_type = data[pos+4:pos+8]
    chunk_data = data[pos+8:pos+8+length]
    pos += 12 + length
    if chunk_type == b'tEXt':
        null_idx = chunk_data.find(b'\x00')
        if null_idx >= 0:
            key = chunk_data[:null_idx].decode('latin-1')
            value = chunk_data[null_idx+1:].decode('latin-1')
            if key == 'chara':
                decoded = base64.b64decode(value)
                parsed = json.loads(decoded)
                d = parsed.get('data', parsed)
                out_path = sys.argv[2] if len(sys.argv) > 2 else 'output.json'
                with open(out_path, 'w', encoding='utf-8') as fout:
                    json.dump(parsed, fout, ensure_ascii=False, indent=2)
                print(f'OK: {os.path.basename(out_path)}')
                sys.exit(0)

print('ERR: no chara chunk found')
sys.exit(1)
```

### 3.2 JSON 处理（纯文件操作，不依赖脚本）

JSON 格式的角色卡可以直接用 `read_file` 读取 → `write_file` 写入：

1. 读取 JSON 文件内容
2. 用 `grep` 或逐行解析提取字段
3. 生成所需文件

**无需 bash 即可完成**（仅限 .json 格式角色卡）。

### 3.3 文件操作（所有平台通用）

| 操作 | 可用工具 | 不依赖 |
|:-----|:---------|:-------|
| 读取文件 | `read_file` | ✅ bash 不需要 |
| 写入文件 | `write_file` / `edit_file` | ✅ bash 不需要 |
| 搜索内容 | `grep` | ✅ bash 不需要 |
| 列出文件 | `glob` / `ls` | ✅ bash 不需要 |
| 复制模板 | `write_file` 重新写入 | ✅ bash 不需要 |
| 创建目录 | `write_file` 自动创建父目录 | ✅ bash 不需要 |

## 五、快速环境检测代码（技能 Step 0 使用）

各技能 Step 0（内部检测步骤）执行以下检测，将结果存入内存变量：

```markdown
### Step 0：环境检测

执行以下检测确定当前平台能力：

1. 尝试 `bash -c "echo env_ok"` → 成功则标记 `env.shell = bash`
2. 尝试 `python --version` → 成功则标记 `env.python = python`
3. 尝试 `node --version` → 成功则标记 `env.node = node`

检测结果决定后续执行路径：
- 有 bash → 全功能模式
- 无 bash 有 python → 写入脚本 + 告知用户手动执行
- 仅文件工具 → 纯读写模式（仅支持 .json 格式角色卡）
```
