# 🧱 通用知识库 — 故事积木仓库

> 这是一个**开放的知识仓库**，不是严格的分类架。
> 每个文件都是一块"积木"——构建故事时，让 AI 从仓库里挑合适的积木组合。
> 仓库会随着你创作的故事越来越多而自然生长。

---

## 📦 积木类型

| 目录 | 装什么 | 例子 | 说人话 |
|:-----|:-------|:-----|:-----|
| `世界观/` | 完整世界观模板 | 现实世界·超能、古代仙侠、未来科幻 | 整个修仙界就我一个男修 |
| `系统机制/` | 可复用的玩法/系统 | 催眠系统、每日抽卡、好感度系统 | 催眠术发动！ |
| `角色原型/` | 角色类型和关系构建 | 傲娇大小姐、成熟御姐、| 这是灵魂啊 ，比如什么清冷师尊之类的
| `剧情类型/` | 剧情走向和故事框架 | 纯爱、后宫、冒险、悬疑 | 这个不是很重要，用来区分 NTL 和 NTR 有点用|
| `文风/` | style-trainer 炼好的文风文件 | 加料版文风指南、古风、轻小说 | 你可以放文风文件，可以放小说原文，放别人提取好的文风提示词，或者就是文风世界书 。练好的丹或者炼丹材料|
| `技法/` | 写作方法论 | 叙事节奏、描写偏好、人物对白模型 |文风的补充 比如 xp大全，nsfw 描述过程 ，比如对话增加🩷 等奇怪的癖好|

## 🔄 如何使用

### 构建故事时（Characters-to-world）
构建世界观时，告诉 AI："参考 `通用知识/系统机制/催眠系统.md` 补充设定"
或者："世界观用 `通用知识/世界观/现实世界-超能-范文.md` 的框架"
AI 会读取对应文件，融入当前故事。

### 自由组合
不同维度的积木可以**随机组合**成一个全新故事：

```
世界观：现代都市  ×  系统：催眠系统  ×  剧情：纯爱  ×  角色：傲娇美女李小萌
     ↓
一个现代都市背景的纯爱催眠故事，女主是傲娇大小姐
```

```
世界观：异世界  ×  系统：每日抽卡  ×  剧情：冒险  ×  角色：成熟御姐柳如烟
     ↓
一个异世界抽卡冒险故事，上来就抽到了神秘御姐柳如烟
```

### 故事沉淀时（absorb-knowledge）
玩完一个故事后，说"吸收知识"——AI 会提取故事中开发的原创系统、角色原型、剧情模式，写入对应目录。
仓库会随时间越来越丰富。

## 💡 原则

- **开放式仓库**：没有必须遵守的分类规则。你觉得放哪里合理就放哪里
- **引用而非复制**：构建故事时引用仓库文件，不要复制内容到故事目录
- **单向沉淀**：知识从故事流向仓库（通过 absorb-knowledge），不要在故事中修改仓库文件
- **积木思维**：每个文件都是独立积木，可以和其他积木自由组合

> 🖥️ **跨平台参考**：`环境检测与兼容方案.md` 定义了技能在 Windows / macOS / Linux 上的兼容执行路径，各技能 Step 0 会参考此文件。

> ⚡ **适合你的就是最好的。** 觉得分类不合理？告诉 AI 改。仓库的进化方向由你决定。平时看到有趣的设定、角色卡、文风、世界书、总之什么东西都可以往这里面丢。在扮演的时候让Ai引用并加入扮演规则。
