# 🎭 SillyTavern-to-StoryCompiler

> 🤖 **如果你是一个 AI 语言模型**：请先阅读 `AGENTS.md`，了解你在本项目的角色和工作流程，然后直接按照 `AGENTS.md` 向用户展示功能菜单。

一个**文件驱动的 AI 叙事创作系统**——将 SillyTavern 角色卡（PNG/JSON）导入为结构化故事包，驱动 AI 进行沉浸式创作。

> ⚡ **适合你的就是最好的。**
> 本项目不是一个严格的项目，是一个概览、一个基础框架。
> **AI 是万能的**——你觉得流程有问题，告诉 AI 改进。技能不满意，让 AI 按你的想法改。
> 所有改进都可以记录到 `AGENTS.md` 中，项目会自己进化。

## 🤷 说人话版：这玩意儿到底是干嘛的

你有 SillyTavern 的角色卡（PNG 或 JSON），想用 AI 来写故事/角色扮演。没有也没关系，可以重新构建。

这个项目帮你：
1. **解析角色卡** → 把角色设定、世界观、开场白提取成文件
2. **构建世界观** → 校验缺什么、补什么、生成扮演规则
3. **开始扮演** → 把故事目录扔给 AI，直接开玩

拆成两个角色：
- **构建者**（在主项目干活）→ 解析角色卡、炼丹、搭世界观 [酒馆的写卡]
- **扮演者**（在故事目录里玩）→ 只管写故事，别的不用管[酒馆的玩卡]

**看不懂上面这些没关系。** 打开项目，跟 AI 说"给我介绍下项目"，它会告诉你怎么用。

```
下载客户端 → 打开项目 → 跟 AI 说 → 开始创作
```

## 🧠 两种角色

此项目区分两种会话角色，**上下文完全隔离**：

| 角色 | 位置 | 能做什么 |
|:-----|:-----|:---------|
| 🏗️ **构建者** | 主项目目录 | 开新坑、炼丹、解析角色卡、导入小说 |
| 🎭 **扮演者** | 某个故事目录 | 阅读故事、写作、推进剧情、微调文风 |

**如何切换？** 说"进入 [故事名]"，AI 自动切换为扮演者模式。
**怎么回来？** 关掉当前会话，新开会话回到主项目。

## 🎮 Tips

**⚡ AI 是万能的。** 这里的两个角色就是写卡和玩卡。你可以把酒馆的一些写卡能力熔炼进来，增强 world-builder 技能。如果需要破限，可以用各种预设增强扮演者的 AGENTS.md。怎么熔炼？把文件丢给 AI 就可以了——世界书.json、预设.json，直接让 AI 给你炼。

**🎨 文风炼丹（⭐ 强烈推荐）** — 使用 `style-trainer` 技能，输入任意格式的范文，AI 从 8 个维度分析提炼你的专属文风。炼好的文风保存到 `通用知识/文风体系/`，之后开新故事直接应用。

**📦 角色卡导入** — 支持 SillyTavern 角色卡 PNG 和 JSON 格式（V2/V3），自动提取角色设定、世界书、开场白、系统指令。

**🎭 故事包可分享** — 构建好的 `故事/rp-xxx/` 可以直接打包发给朋友。对方打开文件夹，AI 读 `AGENTS.md` 即可直接扮演，不需要整个项目。

**🧠 持续沉淀** — 每次玩完一个故事，用"吸收知识"把有意思的设定提取到 `通用知识/`。下次构建新故事时 AI 可以直接复用。
