# archive-story — 故事归档器

## 职责

当一个故事全部完成后，执行以下操作：
1. 将当前 `故事/rp-故事名/` 的完整内容备份到 `故事/归档/{故事名}/`
2. 清理 `故事/rp-故事名/` 目录（删除所有内容）
3. 调用 `absorb-knowledge` 先沉淀知识
4. 更新归档清单

## 触发方式

用户说以下任意一句：
- "归档"
- "完结归档"
- "故事完成，归档吧"
- "存档当前故事"

## 执行流程

### 0. 前提：先沉淀知识

在执行归档前，**必须先调用知识沉淀**（自动触发 `absorb-knowledge`），确保故事中开发的玩法/技巧已被记录到知识库。

### 1. 确定故事名

从以下来源确定故事名称（优先级从高到低）：
1. `故事/rp-故事名/设定/世界背景.md` 第一行标题
2. `故事/rp-故事名/设定/主角.md` + "的世界"
3. 当前日期 + "未命名故事"
4. 询问用户

### 2. 创建归档目录

```bash
mkdir -p 故事/归档/{故事名}
```

### 3. 复制所有内容

将 `故事/rp-故事名/` 下的所有内容复制到归档目录：
```bash
cp -r 故事/rp-故事名/ 故事/归档/{故事名}/
```

### 4. 记录归档元数据

在 `故事/归档/{故事名}/` 下创建 `归档信息.md`：

```markdown
# 归档信息

**故事名称：** {故事名}
**归档日期：** {日期}
**章数：** {N} 章
**总字数：** {估算字数}
**主要角色：** {角色名列表}
**世界观类型：** {奇幻/科幻/历史/都市/悬疑等}
**核心玩法：** {本故事使用的核心玩法列表}
**备注：** {任何补充说明}
```

### 5. 清理故事目录

删除 `故事/rp-故事名/` 下的所有内容：
```bash
rm -rf 故事/rp-故事名/
```

### 6. 提交 git

```bash
git add -A
git commit -m "📦 归档：{故事名}（{N}章）"
```

### 7. 输出归档结果

```
【归档完成】
故事「{故事名}」已归档到 `故事/归档/{故事名}/`
共 {N} 章，约 {字数} 字
知识已沉淀到 `技能/创作资产沉淀库.md`
```

## 重要事项

- **归档后 `故事/rp-故事名/` 完全删除** — 确保用户确认故事已全部完成后再执行
- **归档目录不可被 AI 读取** — `故事/归档/` 已被禁止读取。归档仅供用户手动查看/备份。
- **git 提交** — 归档是一次完整的故事封存，应当提交到 git
