# absorb-knowledge — 知识沉淀器

## 职责

当一个故事完成（或用户说"吸收知识"），提取本故事中开发的原创玩法、调教技巧、体位、叙事结构，追加到 `技能/创作资产沉淀库.md`。

## 触发方式

用户说以下任意一句：
- "吸收知识" / "沉淀知识"
- "提取本故事的可复用内容"
- "把学到的东西记下来"
- 故事完成后自然触发

## 输入

- `故事/rp-故事名/写作/小说/` 下的所有章节文件
- `故事/rp-故事名/状态/` 下的所有状态文件
- `故事/rp-故事名/设定/` 下的设定文件
- `故事/rp-故事名/写作/规划/` 下的规划文件（如有）
- `技能/创作资产沉淀库.md` — 现有知识库（追加模式）

## 执行流程

### 1. 扫描已有知识库

读取 `技能/创作资产沉淀库.md`，了解现有分类体系。

现有分类参考：
- 一、核心玩法类（如蒙眼替换系统、摄像头表演体系等）
- 二、玩法设计类
- 三、体位/场景设计类
- 四、叙事结构类
- 五、风格体系类
- 六、金句/名场面
- 七、创作检查清单

### 2. 分析故事内容

读取以下文件提取可复用的知识：

#### A. 从 `故事/rp-故事名/写作/小说/` 各章节提取：
- **新玩法/系统** — 本故事中首次开发的原创机制
- **新体位** — 有特色的性爱姿势/场景设计
- **新玩法机制** — 本故事首次开发的原创机制
- **叙事技巧** — 视角切换、节奏控制、悬念设置等

#### B. 从 `故事/rp-故事名/设定/` 提取：
- **角色原型** — 可复用的角色类型（不是具体角色，是类型）
- **世界观机制** — 可复用的设定机制

#### C. 从 `故事/rp-故事名/状态/` 提取：
- **事件模式** — 反复出现的有效剧情模式

### 3. 去重

对比已有知识库中的条目，**不要重复添加**已存在的内容。

### 4. 写入沉淀

将新知识追加到 `技能/创作资产沉淀库.md`，格式如下：

```markdown
### {序号}.{子序号} {名称}
**适用场景：** {什么类型的故事适合用}
**核心机制：**
- {机制要点1}
- {机制要点2}
- {机制要点3}
**关键爽点：** {这个玩法为什么爽}
**来源故事：** {故事名称}
```

追加到对应分类下，若无对应分类则在末尾新建分类。

### 5. 总结输出

输出本次沉淀的摘要，格式：
```
【知识沉淀完成】
新增 {N} 条知识：
- 玩法类：{N} 条
- 玩法设计：{N} 条
- 体位/场景：{N} 条
- 叙事结构：{N} 条
知识库位置：`技能/创作资产沉淀库.md`
```

## 注意事项

- **只提取可迁移的知识** — 不要记录具体剧情、角色名、故事线。只沉淀能用到下个故事的通用资产。
- **宁少勿滥** — 只有真正有复用价值的内容才沉淀。
- **保持现有分类风格** — 与已有条目的详细程度和格式保持一致。
