# dm — 剧情方向引导

## 职责

用户给出笼统的写作方向后，将其转化为 2-3 个具体的写作选项。

每个选项只描述方向和关键元素，不展开写。

---

## 输入

来自用户的自然语言方向描述，例如：
- "来一段浴室场景"
- "写一段他们吵架后和好的"
- "推进一下主线"
- "写一段女主勾引男主的戏"

以及当前 `故事/rp-故事名/状态/` 下的剧情上下文。

---

## 输出

格式化的选项列表，例如：

```
根据当前状态，我建议以下方向：

① 【场景】浴室缠绵
   【基调】温柔细腻
   【侧重】情绪铺垫 → 身体接触
   【关键元素】热水蒸汽、肌肤触感、轻声细语

② 【场景】浴室激情
   【基调】热烈急迫
   【侧重】感官冲击 → 高潮爆发
   【关键元素】水声、喘息、失控感

③ 【场景】浴室对峙
   【基调】张力紧张
   【侧重】权力博弈 → 心理攻防
   【关键元素】镜面反射、冷热水交替、沉默

选一个，我展开写。
```

---

## 选项设计规则

### 选项数量
- 通常 3 个
- 用户明确说"随便"时，可以只出 2 个或 1 个默认选项

### 选项差异
- 每个选项的情绪基调必须不同
- 节奏/侧重点必须不同
- 让用户有真正的选择感

### 选项维度

从以下维度中选 3-4 个来设计差异：

| 维度 | 可选值 |
|------|--------|
| 情绪基调 | 温柔/粗暴/急迫/缓慢/甜蜜/虐心/紧张/放松 |
| 节奏 | 快速/中速/缓慢/有起伏 |
| 侧重点 | 情绪/感官/剧情/对话/心理/动作 |
| 权力动态 | 支配/服从/平等/对抗 |
| 性爱侧重 | 前戏/正戏/后戏/全程 |
| 镜头焦点 | 全景/特写/切换 |
| 关键元素 | 具体意象（水声/烛光/月色/汗水/……） |

### 特殊处理

**用户说"继续"**：读取 `故事/rp-故事名/状态/事件日志.md` 最后一条，推断自然发展方向

**用户说"自由发挥"**：基于当前剧情张力最大的方向出选项

**用户说"按我们之前说的"**：查 `故事/rp-故事名/状态/` 看有没有未完成的伏笔或计划

---

## 读取要求

出选项前必须读取：
1. `故事/rp-故事名/状态/世界概览.md` — 当前时间/地点
2. `故事/rp-故事名/状态/角色状态.md` — 角色当前状态
3. `故事/rp-故事名/状态/关系状态.md` — 角色间关系
4. `故事/rp-故事名/状态/时间线.md` — 最近发生的事件
5. `故事/rp-故事名/风格/风格分析/风格摘要.md` — 文风速查
