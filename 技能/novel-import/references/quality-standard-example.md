# 质量标准参考

用户认可的高质量输出示例来自《女神堕落系统》提取数据。

## 用户反馈

用户明确表示喜欢"一开始的效果"（8个详细文件），不满意后来"加料版"（只有4个文件）。

## 质量要点

### 角色.md
- 每个角色必须有：身份、阵营、性格、能力、主要关系、当前状态
- "当前状态"必须用原文细节，不能笼统描述
- 示例：
  ```
  # 杨昊然
  身份：16岁高中生，玉林高中高一（5）班，堕天使游戏掌控者
  阵营：主角方
  性格：玩世不恭、好色、占有欲极强、聪明但不爱学习
  能力：堕落戒指赋予的学习药剂效果（超强记忆力/目力）
  主要关系：柳若曦（母亲/母犬）、杨梦瑶（双胞胎妹妹/奶牛）...
  当前状态：期末考试结束，父亲已搬出家中，准备接管家庭。宠物庄园改造中。
  ```

### 剧情状态.md
- 必须包含"各角色最终状态"表格（角色/身份/状态 三列）
- 必须有当前主线/支线/冲突/目标/敌人/盟友
- 示例表格：
  ```
  | 角色 | 身份 | 状态 |
  |------|------|------|
  | 杨昊然 | 掌控者 | 期末考结束，准备接管家庭 |
  | 柳若曦 | 母亲 | 怀孕，离婚，关系复杂化 |
  ```

### 系统设定.md（系统文必须）
- 必须列出所有属性名+含义+数值范围
- 必须有任务系统详细记录（任务名/触发条件/奖励）
- 必须有商城/资源系统
- 必须有特殊规则和已知漏洞

### 伏笔库.md
- 必须区分未回收/已回收
- 标注出现章节和回收章节
- 示例：
  ```
  - 肖少婉的网名"邪"的崇拜者 → 杨昊然就是"邪"（第24章伏笔，第41章回收）
  ```

### 导演笔记.md
- 必须有"不建议触碰的设定"
- 必须有"风险点"
- 续写方向要具体，不能泛泛而谈

## 反面教材

"加料版"只有4个文件（角色/世界背景/剧情状态/章节索引），缺少势力/地点/时间线/伏笔库/导演笔记，质量差距明显。
