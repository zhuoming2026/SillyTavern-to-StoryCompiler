# novel-import

## 职责

从完整小说中提取：世界观、角色、势力、时间线、当前剧情状态、未回收伏笔、续写所需资料。
生成标准 Story Engine 项目，让 AI 能在不了解原著的情况下继续创作。

禁止：续写小说、总结章节、评价作品。

## 输入

支持单文件（.txt/.md）或多文件（每章一个文件）。

## 工作流程

### STEP 1: 统计规模 + 切分章节

1. 用 `wc -c` 检查文件大小。超过 500KB 必须切片处理。
2. 写 Python 脚本切分章节（见 `scripts/split_chapters.py`）。
3. 中文小说常见章节标记格式需全部兼容：
   - `第X章`（阿拉伯数字）
   - `第X章`（中文数字：第一章、第二十章、第七十章）
   - `第 X 章`（数字前后有空格）
   - `序章`、`前言`
4. 章节标记可能嵌在行中间（非行首），需要同时检测行首和行内模式。

**PITFALL**: 中文数字到阿拉伯数字的映射要覆盖到至少"七十"（70），否则前期章节全部丢失。

### STEP 2: 建立中间目录

```
分析缓存/
├── 章节文件/        # 切分后的单章文件
│   └── _index.txt   # 章节索引（标题|文件名|行数|起始行）
├── 批次文件/        # 合并后的批次文件（每批~50章）
├── 精简章节/        # 精简版（每章提取开头/结尾/对话/角色）
├── 提取结果/        # 程序化提取的 JSON
章节摘要/
角色发现/
势力发现/
地点发现/
伏笔发现/
提取数据/            # 最终输出
```

### STEP 3: 程序化预提取（Phase 1）

写 Python 脚本扫描全文，提取：
- 角色名出现频次（预定义已知角色列表 + 从"X说/道"模式自动发现）
- 地点关键词频次
- 系统/世界观关键词频次
- 章节标题列表

输出 JSON 到 `分析缓存/提取结果/phase1_extraction.json`。

**PITFALL**: 自动发现的角色名会混入大量噪声词（"不知"、"忍不住"、"或者"、"轩突然"、"并没有"）。需要设置停用词表和最低频次阈值（建议 ≥10）。**更重要的是**：中文小说的正则角色名提取本身不可靠——"XX说/道"模式会匹配到大量非人名组合。程序化方法只适合做初筛（找高频候选），最终角色列表必须人工阅读关键章节确认。对于 2MB+ 的小说，直接读首/中/末三段关键章节 + 搜索系统面板条目，比跑正则更高效。

### STEP 4: 创建精简章节

为每章提取：
- 出现的角色名（从已知列表匹配）
- 开头 5 行（场景/设定）
- 关键对话（引号内容，最多 5 条）
- 结尾 3 行（悬念/转折）

合并为 3 个阶段文件（early/middle/late），每个控制在 50-70KB。

### STEP 5: 深度分析

**关键决策：是否使用子代理？**

- 如果模型是 Claude/GPT-4 级别且 max_concurrent_children ≥ 3：用 delegate_task 并行分析 3 个阶段
- 如果模型是较小模型（如 mimo）或并发限制低：**不要委托子代理**，直接在主会话中逐段读取精简文件并写输出

**PITFALL (mimo 模型)**: 通过 delegate_task 委托给 mimo 等较小模型时，子代理倾向于只读文件或说"我将开始..."然后立即返回，不实际执行写文件操作。如果子代理返回后 `find ./分析缓存 -name "*.md"` 无输出，说明子代理未工作，必须改为自己处理。

每阶段提取：
- 角色信息（首次出现、身份、性格、关系、重要事件、当前状态）
- 势力/组织
- 地点
- 伏笔（已回收/未回收）
- 每章事件摘要

### STEP 6: 分析最终状态

重点分析最后 20 章。提取：
- 当前主线/支线/冲突
- 当前地点/目标/敌人/盟友
- 各角色最终归宿
- 大结局内容

### STEP 6.5: 系统设定提取（系统文/玄幻文必须）

当小说含有以下元素时，必须额外生成 `系统设定.md`：
- 游戏系统（属性面板、任务、商城）
- 修炼体系（境界、功法、灵力）
- 超能力（异能、血脉、天赋）
- 等级制度（星级、段位、品阶）

**系统设定比世界观更重要**，优先级：系统 > 角色 > 剧情 > 世界观。

提取内容：
1. 系统来源/激活方式
2. 核心机制（运行规则、限制条件）
3. 属性面板（所有属性名+含义+数值范围+成长方式）
4. 技能/职业体系（分类、特性、转职条件）
5. 任务系统（类型、触发、奖惩）
6. 商城/资源系统（货币、商品、关键道具）
7. 特殊规则（独有机制、副作用）
8. 系统漏洞/未解之谜

### STEP 6.6: 特殊题材素材提取（如有）

当小说含有特定题材元素时，可额外生成题材分析文件：
- 特殊权力关系、奴役契约
- 惩罚、体罚、羞耻play
- 道具使用、束缚、捆绑
- 角色扮演、制服play

**提取必须详细到可直接复用**，不能只列关键词，要写清楚具体操作方法。

#### 必须提取的 5 个模块

**模块一：道具库**（按类别分组，每种道具写明名称/用途/使用场景/注意事项）

| 类别 | 必须覆盖的道具 |
|------|---------------|
| 束缚类 | 项圈、口球/口枷、手铐、脚镣、颈手枷、眼罩、绳索、铁链、束缚带 |
| 插入类 | 振动棒、跳蛋、假阳具、肛塞（充气/固定）、双头龙、珠串、尿道塞、贞操带 |
| 乳房类 | 乳夹、乳环、束胸、催乳剂/催奶针、挤奶器 |
| 纹身标记 | 淫纹（位置/内容/颜色）、标签、挂牌 |
| 制服类 | 女仆装、猫女郎/兔女郎、校服、护士服、空姐服、旗袍、丝袜+高跟 |
| 药物类 | 催乳剂、春药、润滑液、灌肠液 |
| 其他 | 蜡烛（低温）、皮鞭、拍板、夹子 |

```

```

**模块二：典型场景**（每种写明环境/互动方式/心理要点）

必须覆盖的场景类型：
- 日常相处（家中/社交场所）
- 室外场景（校园/商场/公园/餐厅）
- 公开场合（直播/舞台/集会）
- 监控视角（摄像头+实时观察）
- 角色扮演（身份扮演/情境演绎）
- 奴隶对战（双头龙对战/地位竞争）
- 惩罚场景（打屁股/打奶光/皮鞭/滴蜡/灌肠/禁食）

搜索关键词：
```

圣女|骑士|偶像|师生|主仆|奴隶对战|双头龙|打屁股|打奶光|灌肠|
滴蜡|皮鞭|禁食|罚站|跪罚|关笼|狗笼
```

**模块三：体位与性爱技巧**（每种写明名称/操作方法/适用场景/配合道具）

必须覆盖的体位：
- 传教士/正常位
- 后入/狗交位
- 骑乘位（女上位）
- 侧入位
- 站立位
- 坐位（椅子/沙发/桌边）
- 跪趴位
- 悬挂位（捆绑后）

搜索关键词：
```

对视|低语|牵手|拥抱|依偎|
内射|颜射|口射|吞精|潮吹|喷水
```

**模块四：心理博弈技巧**（每种写明原理/步骤/效果/风险）

必须覆盖：
- 奴性开发阶段（从抗拒到主动渴望的渐进过程）
- 称呼体系（自称/他称的等级划分）
- 请安/仪式感（跪拜/叩首/换鞋等日常仪式）
- 关系日记（角色关系变化记录制度）
- 把柄控制（照片/视频/契约/社交账号）
- 羞耻感利用（公众身份/社会关系/暴露风险）

**模块五：奴隶管理体系**（如有）

必须覆盖：
- 奴隶契约模板（完整条款）
- 奴隶等级体系（等级划分/权限/义务）
- 散养/圈养分类（自由度/监控要求/汇报频率）
- 饮食/着装/行为规范

### STEP 7: 生成输出文件

```
提取数据/
├── 世界背景.md      # 世界概况/力量体系/社会结构/势力格局/特殊规则
├── 系统设定.md      # 【系统文必须】系统来源/属性/技能/任务/商城/特殊规则
├── 题材分析.md # 【特殊题材】题材专项分析文件
├── 角色.md          # 按重要度排序，含身份/性格/能力/关系/当前状态
├── 势力.md          # 势力名/性质/目标/盟友/敌人/当前状态
├── 地点.md          # 地点/位置/功能/关联剧情
├── 时间线.md        # 按卷/阶段分组的事件列表
├── 剧情状态.md      # 最重要！当前章节/时间/地点/主线/支线/冲突/目标
├── 伏笔库.md        # 未回收/已回收两部分
└── 导演笔记.md      # 续写方向/风险点/禁区
```

## 输出优先级

剧情状态 > 角色状态 > 世界背景

重点保证：读完这些文件后，另一位 AI 能从当前章节继续写下去。

## 大文件策略

超过 100 万字时：
1. 章节扫描（程序化）
2. 缓存摘要（精简版）
3. 汇总分析（LLM）

禁止依赖单次分析。

## 支持文件

- `scripts/split_chapters.py` — 章节切分脚本，兼容多种中文章节标记格式
- `references/subagent-delegation-pitfalls.md` — 子代理委托踩坑记录（mimo等小模型不执行任务的问题及解决方案）
- `references/quality-standard-example.md` — 用户认可的高质量输出示例和反面教材
- `references/system-novel-extraction.md` — 系统文提取技巧（属性面板搜索、女性专属属性、特性记录）
