# Characters-to-world — 故事构建器

## 职责边界

**在角色卡导入完成后接手（依赖角色卡导入的三个产出文件）：**

- `设定/角色卡原始数据.json`
- `设定/世界书概览.md`（用户已打勾）
- `设定/🎬 开场白.md`

**不做的事情：**
- ❌ 不提取原始数据（那是 importer 的活）
- ❌ 不做世界书概览（那是 importer 的活）
- ❌ 不替用户判断——展示、提问、等回答

---

## 工作流

### Step 0：环境检测

Characters-to-world 所有步骤均为纯文件操作（读 JSON → 分析 → 写设定文件），**不依赖 bash 或脚本执行**，在所有平台通用。

无需特别适配。如果 `bash` 不可用，不影响本技能执行。

---

## 工作流

### Step 3：读取用户勾选 → 生成全部设定文件

#### 3a. 读取用户勾选

读取 `设定/世界书概览.md`，解析 `[x]` 标记的条目 ID。
从 `设定/角色卡原始数据.json` 中提取对应的完整条目内容。

如果用户还没打勾 → 提示用户先打勾再继续。

#### 3b-1. 硬提取（脚本执行，不经过 AI）

根据条目的 `comment` 和 `keys` 将勾选条目原样写入对应文件：

| 分类 | 判断依据 | 输出文件 |
|:-----|:---------|:---------|
| 世界观 | 涉及场景/系统/设定/规则/玩法 | `设定/世界背景.md`（原始草稿） |
| 角色 | 涉及角色卡/人物描述 | `设定/角色.md`（原始草稿） |
| 主角 | 涉及 {{user}}/主角/竿役 | `设定/主角.md`（原始草稿） |
| 风格/变量 | 风格指南条目 / MVU 变量条目 | 单独处理 |

**同时产出硬数据文件（模式匹配，不经过 AI）：**
- `设定/角色硬数据.md` — 名称、年龄、外貌、固定关系等可模式匹配的事实

**规则：** 硬提取只做原样复制，不做内容修改或 AI 补充。
**注意：** 原始草稿文件内容可能是混合的（一个条目同时包含世界观和角色信息），这是正常的——下一层 AI 骨架归纳会进行拆分重组。

#### 3b-2. AI 骨架归纳（有固定结构约束）

> 此步骤在硬提取完成后执行，目的是将混合的原始条目拆分为结构化的设定文件。
> AI 在固定骨架内填充内容，不得擅自增减章节，不得删除原文中的任何事实。

**输入：** 3b-1 产出的原始草稿文件 + `设定/角色卡原始数据.json`
**锚点要求：** 每项归纳结果必须引用原文行号（见下方示例）

**输出文件与固定骨架：**

| 文件 | 固定骨架 | 说明 |
|:-----|:---------|:------|
| `设定/世界背景.md`（最终版） | 世界类型/时代背景 → 核心规则/特殊设定 → 势力/家族/组织 → 当前时间点/事件起点 | 覆盖原始草稿 |
| `设定/角色.md`（最终版） | 每个角色：名称/身份 → 外貌特征 → 性格特质(引用行号) → 动机/目标 → 关系 → 原文附录 | 覆盖原始草稿 |
| `设定/主角.md`（最终版） | 名称/身份 → 玩家可定 → 故事已固定 → 多层性格(如有) | 覆盖原始草稿 |

**行号引用示例（必须在每个归纳项中标注来源）：**

```markdown
## 角色：苏婉清

- **身份**：新娘，苏家长女（来源：设定/角色卡原始数据.json #12）
- **性格**：温婉贤淑，但内心有自己的坚持（来源：设定/角色卡原始数据.json #15-18）
  - 原文摘要："苏婉清看似温柔顺从，实则有自己的坚持……"
- **多层性格**（如果存在）：
  | 层面 | 核心特征 | 激活条件 |
  |:-----|:---------|:---------|
  | 得体妻子 | 温柔体贴、懂礼数 | 默认启用 |
  | 习俗渐放开 | 羞涩但配合 | 完成一次压床后 |
  | 真心层 | 卸下伪装 | 亲密度 ≥ 50 |
```

**约束规则：**
1. 不得删除原始数据中的任何事实（可以在骨架中重组，不得删除）
2. 不得擅自增减骨架章节（固定结构不可改动）
3. 每条归纳必须标注来源行号
4. AI 推断的内容必须标注"（AI 推断）"以区别于原文
5. 如果原始数据无法填充骨架中的某个字段 → 留空并标注"（原始数据未提供）"

**关于多层性格：** AI 从原始数据中识别角色是否有多个性格层面（如人前 vs 人后、前期 vs 后期、特定关系下的特殊态度）。如果有，在角色条目中列出层面表；如果只有一个层面，则不需要。

#### 3c. 提取风格指南

**优先级：**
1. 用户勾选的条目中，涉及「文风/叙事/描写/风格/用语」的条目（最高优先）
2. 开场白 `first_mes` 的语言风格（次优先）
3. `mes_example` 的对话风格（补充参考）

**没有文风条目时**，从开场白中提取风格特征：
- 叙事基调与氛围
- 叙事视角
- 语言风格特征（句式、修饰、感官侧重）
- 各角色语言特征（从对话中提取典型口吻和用词）
- 来源说明

**产出：** `设定/风格指南.md`

#### 3d. 提取变量系统（MVU 系统 ★）

**核心入口：** 检查用户勾选的条目中，`comment` 含「更新规则」且 `content` 含 `<UpdateVariable>` 的条目。

如果存在，提取完整的 MVU 系统到 `设定/📊 状态变量.md`：

```
| 变量 | 说明 | 类型 | 更新规则 |
```

包含：
- 变量定义表
- 序列等级对照表（如有）
- 强制更新法则（逐条提取，原样保留）
- 输出格式示例（_.set() 调用格式）

⚠️ **特别注意：** MVU 系统的核心是 `<UpdateVariable>` + `_.set()` + 强制更新法则。提取时保留完整的格式规范（纹章 JSON 嵌套格式、数值纯数字要求等）。

如果找不到 MVU 条目 → 空文件，标注"未检测到变量系统"。

#### 3e. 提取思维链（如果有）

**核心入口：** 检查勾选的条目中是否有涉及「思考步骤/写作流程/叙事步骤/内部流程」的内容。

如有 → 提取并写入 `设定/思维链.md`
如无 → **不生成空文件**，AGENTS.md 中使用通用模板

### Step 4：组装 AGENTS.md（v2.1 三合一公式）

> 公式：**AGENTS.md = 核心规则 + 预设规则 + 故事骨架**
> 文风公式：**#4 写作引擎 = 原版角色卡 + 预设文风 + 用户要求**
> 冲突时优先级：用户要求 > 预设文风 > 原版角色卡

#### 4a. 读取三段组件

| 段 | 文件 | 提供内容 | 占位符处理 |
|:--|:-----|:---------|:----------|
| ① 核心规则 | `模板/扮演者-核心规则.md` | #0 #1 #4.3 #4.4 #4.5 #7 #8 #9 #13 #14 | 替换 `{{STORY_NAME}}` |
| ② 预设规则 | `模板/预设/扮演者-[名]-规则.md`（如有） | 身份替换 + 语料库填充 + 纠错追加 + 思维链 | 替换 ① 的 `{{IDENTITY_PRESET}}`；填充 `{{CORPUS_OVERRIDE}}`；追加 `{{STYLE_FIX_OVERRIDE}}` |
| ③ 故事骨架 | `模板/故事/AGENTS.md` | #2 #3 #4.1 #4.2 #5 #6 #10 #11 #12 附录 | 按映射表填充 |

#### 4b. 组装流程

```
Step 4b-1: 读取 ① 模板/扮演者-核心规则.md
           ├─ 替换 {{STORY_NAME}} → 故事名
           └─ 保留 {{IDENTITY_PRESET}} {{CORPUS_OVERRIDE}} {{STYLE_FIX_OVERRIDE}} {{STORY_SPECIFIC_BANS}}

Step 4b-2: 如有预设文件，读取 ② 模板/预设/扮演者-[名]-规则.md
           ├─ "身份覆盖" → 替换 ① 整个 #0（含 {{IDENTITY_PRESET}} 占位符）
           ├─ "语料库" → 填充 ① 的 {{CORPUS_OVERRIDE}}
           └─ "文风纠错追加" → 替换 ① 的 {{STYLE_FIX_OVERRIDE}}
           如无预设 → 删除 {{IDENTITY_PRESET}} 行，{{CORPUS_OVERRIDE}} 留空

Step 4b-3: 读取 ③ 模板/故事/AGENTS.md 故事骨架
           └─ 按 4c 映射表填充所有故事特定占位符

Step 4b-4: 合并 ① + ③ → 写入 故事/rp-xxx/AGENTS.md
```

#### 4c. 占位符映射

| 占位符 | 所属 | 来源 | 说明 |
|:-------|:-----|:-----|:-----|
| `{{STORY_NAME}}` | ①+③ | 目录名 | 去 rp- 前缀 |
| `{{IDENTITY_PRESET}}` | ① | 预设 | **替换**整个 #0 |
| `{{CORPUS_OVERRIDE}}` | ① | 预设/角色卡 | **填充** #4.3（默认为空） |
| `{{STYLE_FIX_OVERRIDE}}` | ① | 预设 | 追加到 #4.4 |
| `{{STORY_SPECIFIC_BANS}}` | ① | 角色卡 | 追加到 #4.5 |
| | | | |
| `{{NARRATIVE_TONE}}` | ③ | 原版+预设+用户 | 文风来源标注在注释中 |
| `{{LANGUAGE_FEATURES}}` | ③ | 原版+预设 | 同上 |
| `{{CHARACTER_VOICES}}` | ③ | 角色卡 | 对话风格 |
| `{{NSFW_RULES}}` | ③ | 角色卡 | 成人内容规则 |
| `{{WORLD_SUMMARY}}` | ③ | 世界背景 | ≤300字 |
| `{{CHARACTER_TABLE}}` | ③ | 角色.md | ≤10人 |
| `{{CURRENT_STATE}}` | ③ | 状态变量 | 初始化 |
| `{{STORY_BRIEF}}` | ③ | 开场白 | 一句话 |
| | | | |
| `{{USER_RULES}}` | ③ | 🔧用户规则（最高优先级） | 统一插槽：色色/金手指/等级/特殊约束等 |
| `{{STORY_BRIEF}}` | ③ | 开场白 | 一句话 |

#### 4d. 生成支持文件

| 文件 | 生成方式 |
|:-----|:---------|
| `故事绑定.md` | 模板 + 填入故事名 |
| `README.md` | 模板 + 填入故事名和简介 |
| `元数据/故事元数据.md` | 自动生成初始进度 |
| `状态/世界概览.md` | 从 #6 状态变量初始化 |
| `状态/角色状态.md` | 从 #6 状态变量初始化 |
| `状态/事件日志.md` | 空模板 |
| `写作/聊天记录/` `写作/小说/` `写作/规划/` | 空目录 |

> ⚠️ 不再生成：`配置/扮演规则.md` `风格/README.md` `状态/状态机.md` `状态/变量定义.md`（已整合）

#### 4e. `{{user}}` 处理规则

- 所有原始数据中的 `{{user}}` **一律保留不替换**
- AGENTS.md 中标注用户输入角色名

### Step 5：预设/文风选择 → 重新组装 → 交互确认

> Characters-to-world 在生成 AGENTS.md 初稿后，必须询问用户预设和文风选择。

#### 5a. 检查可用预设

扫描 `模板/预设/` 目录，列出可用的 `扮演者-*-规则.md` 文件。

#### 5b. 展示选择

```
📖 故事「XXX」已生成初稿！

在最终确认前：

🎭 预设规则（可选）：
   可用预设：
   1. TGbreak V3.0.8 — 纠错/防抢话/COT思维链/瑟瑟多样化
   2. 双人成行·青云上 — Atri & Deach 双生写手人格
   3. 不使用预设
   
   请选择预设编号，或说"跳过"。

🎨 文风调整（可选）：
   - 当前文风按角色卡开场白提取
   - 想补充什么风格？（如：搞笑网文、古风、轻小说...）
   - 或说"跳过"。
```

#### 5c. 根据选择重新组装

用户选择后，重新执行 Step 4 组装流程，注入预设和/或文风调整。

#### 5d. 呈现最终概览 + 确认

```
📖 故事「XXX」已生成！

• 主角：{{user}}（占位符，请告诉我你的名字）
• 世界观：按角色卡
• 文风：原版角色卡风格 + [预设文风/用户调整]
• 预设规则：[已注入 XXX] 或 [无]
• 变量系统：MVU 已就绪（如有）

有什么想改的吗？没有就说"好了"进入故事。
```

用户每说一项，当场修改对应文件。全改完后，用户说"好了"或"进入故事"，进入 5e。

#### 5e. 完成确认

```
✅ 故事「XXX」构建完成！

说"进入 XXX"开始扮演，或继续修改设定。
```

## 依赖

**必须存在以下文件才能工作：**
- `设定/角色卡原始数据.json`
- `设定/世界书概览.md`（用户已打勾）
- `设定/🎬 开场白.md`

**如果缺失** → 提示用户先调用 SillyTavern-Characters-importer 导入角色卡。

## 绝对禁令

- ❌ 禁止替用户做默认判断——默认按角色卡走，用户不改就不改
- ❌ 禁止跳过交互环节
- ❌ 禁止对硬提取数据（3b-1 产出）做 AI 修改——保持原样，禁止"推断"和"补充"
- ❌ 禁止 AI 骨架归纳（3b-2）跳过约束规则（行号引用、固定骨架、原文事实保留）
- ❌ 禁止替用户做勾选——用户不勾不能继续
- ❌ 禁止在提取 MVU 时遗漏更新规则和格式要求
- ❌ 禁止在 AGENTS.md 中大段复制原始数据（必须使用精简表格）
- ❌ 禁止使用通用思维链模板——必须根据故事上下文定制
