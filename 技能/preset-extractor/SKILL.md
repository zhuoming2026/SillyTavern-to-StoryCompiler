# preset-extractor — 预设提取器

## 职责边界

从 SillyTavern 预设 JSON 中提取提示词，经用户勾选确认后，生成**一个**可复用的 `扮演者-[预设名]-规则.md` 文件。

**输入：** 酒馆预设 JSON 文件
**输出：** `模板/预设/扮演者-[预设名]-规则.md`

---

## 工作流

### Step 1：解析 → 三区分类 + 提取文本

读取 JSON 的 `prompts[]` 数组，对每个条目同时做三件事：**分类 + 提取文本 + 清理宏**。

#### 1a. 分类规则（按优先级匹配，命中即停）

| 优先级 | 判断条件 | 分类 | 注入位置 | 说明 |
|:------:|:---------|:-----|:--------|:-----|
| 0 | `name` 含 "⚠️" / "别关" / "&&" / "Chat Examples" | 🚫 纯前端 | — | 使用说明/宏触发/示例 |
| 0 | `content` 仅为 `{{setvar::}}` 或 `{{getvar::}}` 宏（无实际文本） | 🚫 纯前端 | — | 纯宏变量 |
| 0 | `content` 为 Regex/HTML/CSS/JS 代码 | 🚫 纯前端 | — | 前端渲染脚本 |
| 1 | `system_prompt: true` 且 content 含身份描述 | 🎭 角色灵魂 | #0 | 覆盖默认身份 |
| 1 | `content` 含 `<system_context>` | 🎭 角色灵魂 | #0 | |
| 2 | `content` 含 `<core_rules>` 或核心法则文本 | [规则] | #1 | |
| 2 | `content` 含 "用户描写权限" / `<user_boundary>` | [规则] | #1 | 防抢话类 |
| 2 | `content` 含 "防全知" / `<Anti-Omniscience>` | [规则] | #1 | 信息隔离类 |
| 3 | `content` 含 "剧情设计" / "重构现状" / "性格分析" / "剧情推理" 等 COT 特征 | [思维链] | #7 | 基础思维链 |
| 3 | `content` 含 "情欲描写强化" / "确定阶段" / "感官聚焦" 等瑟瑟COT特征 | [思维链] | #7 | 瑟瑟思维链 |
| 3 | `content` 含 "变量更新协议" / "强制状态栏盘点" / "逐项逼问" | [思维链] | #7 | 变量协议 |
| 3 | `content` 含 "世界书审查" / "宏观俯瞰" / "抽丝剥茧" | [思维链] | #7 | 防不读世界书 |
| 3 | `content` 含 "活人感" / "性格延伸" / "人性复杂" | [思维链] | #7 | OOC 避免 |
| 4 | `name` 含 "文风" / "style" / `content` 含 `<wfeeling>` | [文风] | #4.1 | |
| 5 | `name` 含 "纠错" / "禁止" / "八股" / "比喻" | [纠错] | #4.4 | |
| 6 | `content` 含身体部位/拟声词/淫语表/色情多样性 | [语料库] | #4.3 | |
| 7 | `content` 含 `<UpdateVariable>` | [变量] | #6 | |
| 8 | 其他有实际提示词文本的条目 | [其他] | 待定 | 用户自行判断 |
| 9 | 以上都不匹配 | 🚫 纯前端 | — | 包装标签/系统占位等 |

#### 1b. 文本提取与清理

对每个归入 🎭 [规则] [思维链] [文风] [纠错] [语料库] [变量] [其他] 的条目：

1. 提取 `content` 字段完整文本
2. 清理酒馆特有宏：
   - 删除 `{{setvar::...}}` `{{getvar::...}}` （宏引用）
   - 删除 `{{//...}}` （注释）
   - 删除 `{{random::...}}` （随机宏）
3. 清理 XML 包装标签（保留内部文本）：
   - `<system_context>...</system_context>` → 提取内部
   - `<core_rules>...</core_rules>` → 提取内部
   - `<wfeeling>...</wfeeling>` → 提取内部
   - `<anti_cliche>...</anti_cliche>` → 提取内部
   - 其他已知 XML 标签同理
4. 保留 `{{user}}` 和 `{{char}}` 不替换
5. 压缩多余空行（3+ → 2）

#### 1c. 互斥检测

同分类下检测功能重叠：
- [文风] 类：多个文风条目标注 `（多选会叠加效果）`
- [规则] 类：防抢话 vs 话痨抢话 标注 `（互斥）`
- [思维链] 类：不检测互斥（可叠加）

---

### Step 2：写入原始清单 → 用户编辑

将分类结果写入 `模板/预设/[预设名]-原始清单.md`。文件分为三个区域：

**区域结构：**
```
🎭 角色灵魂          — 身份描述文本
📋 可直接注入        — [规则] [思维链] [文风] [纠错] [语料库] [其他]
📋 思维链完整文本    — 对应上方 [思维链] 勾选条目的完整提示词
🚫 纯前端渲染        — 仅列名称 + 原因
```

**勾选格式：**
- `[x]` 默认勾选（来自 JSON `enabled: true`）
- `[ ]` 默认不勾选（来自 JSON `enabled: false`）
- 每个条目格式：`- [x] 条目名 — 一句话说明`

**思维链完整文本区：**
- 仅列出上方 [思维链] 分类的条目
- 每个条目附完整清理后的提示词文本
- 标注"可提取到 AGENTS.md #X 的内容"

---

### Step 2：写入原始清单 → 用户编辑

将三区分类结果写入 `模板/预设/[预设名]-原始清单.md`，**不在此步骤交互**。

#### 原始清单文件格式（三区结构）

```
🎭 角色灵魂          → 身份描述文本
📋 可直接注入        → [规则] [思维链] [文风] [纠错] [语料库] [其他] 分类勾选列表
📋 思维链完整文本    → 对应 [思维链] 条目的完整提示词 + 注入建议
🚫 纯前端渲染        → 仅列名称 + 原因，无勾选框
```

**勾选格式：** `[x]` = JSON `enabled:true`，`[ ]` = `enabled:false`
**思维链完整文本区：** 每个思维链条目附完整清理后的提示词，标注"可提取到 AGENTS.md #X"

#### 写入规则

1. 按 1a 优先级分类所有条目
2. 对每个非🚫条目提取并清理 `content` 文本
3. **勾选状态** = JSON 的 `enabled` 字段
4. **分类排序**：规则 → 思维链 → 文风 → 纠错 → 语料库 → 其他
5. **思维链区**：附完整文本 + "可提取到 AGENTS.md #X" 标注
6. **🚫区**：仅列名称 + 原因，不提取文本

#### 完成后提示

```
✅ 已生成 模板/预设/[预设名]-原始清单.md

请打开此文件，将 ☐ 改为 ☑ 勾选你需要的项目。
编辑完成后，在对话中说"生成 [预设名]"继续。
```

---

### Step 3：读取清单 → 生成预设规则

当用户说"生成 [预设名]"时：

1. 读取 `模板/预设/[预设名]-原始清单.md`
2. 解析所有 `[x]` 标记的条目
3. 从原始 JSON 中提取对应条目的 `content`，清理 XML 标签和酒馆宏
4. 按四段式结构生成 `模板/预设/扮演者-[预设名]-规则.md`

```markdown
# 扮演者预设：[预设名]

> 来源：[预设文件名]
> 提取时间：[日期]

---

## 身份覆盖

[替换 AGENTS.md #0 的默认身份]

[从"角色灵魂"提取，清理后的纯文本]

---

## 规则追加

[以下章节在 AGENTS.md 中对应的位置追加内容]

### 追加到 #7 写作流程（思维链）

[用户选中的思维链条目内容，清理宏引用]

### 追加到 #4.1 叙事基调

[用户选中的文风预设内容]

### 追加到 #4.3 语料库

[用户选中的语料库追加内容]

### 追加到 #4.4 文风纠错

[用户选中的纠错规则]

---

## 注入映射

| 本章节 | 注入 AGENTS.md 位置 | 方式 |
|:-------|:-------------------|:-----|
| 身份覆盖 | #0 你是谁 | 替换默认身份段落 |
| 追加到 #7 | #7 写作流程 | 增强步骤①-④的具体检查项 |
| 追加到 #4.1 | #4.1 叙事基调 | 追加在故事文风之后 |
| 追加到 #4.3 | #4.3 语料库 | 追加在通用语料库之后 |
| 追加到 #4.4 | #4.4 文风纠错 | 追加在通用纠错表之后 |
```

写入 `模板/预设/扮演者-[预设名]-规则.md`。

---

### Step 4：提示使用

```
✅ 已生成 模板/预设/扮演者-[预设名]-规则.md

在 Characters-to-world 构建新故事时，本文件将自动注入 AGENTS.md：
- 身份覆盖 → 替换 #0 默认身份
- 规则追加 → 注入 #4.1 #4.3 #4.4 对应位置
```

## 依赖

- 输入：酒馆预设 JSON 文件（`.json`）
- 输出目录：`模板/预设/`

## 绝对禁令

- ❌ 禁止提取 Regex 脚本、HTML/CSS、UI 配置
- ❌ 禁止覆盖已有的同名预设文件（提示用户确认）
- ❌ 禁止修改原预设 JSON 文件
- ❌ 禁止跳过用户勾选步骤（必须人工确认）
