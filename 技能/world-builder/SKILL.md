# world-builder — 世界观设定引导

## 职责

引导用户逐项填充 `故事/rp-故事名/设定/` 下的故事设定文件。

## 工作流

### 1. 检查现状

读取 `故事/rp-故事名/设定/` 目录，检查以下 5 个文件是否存在且有内容：

| 文件 | 必需？ | 通常内容 |
|------|--------|---------|
| 世界背景.md | ✅ | 世界观、等级制度、社会规则、势力分布 |
| 角色.md | ✅ | 所有 NPC 档案 |
| 主角.md | ✅ | 主角身份信息 |
| 风格指南.md | ✅ | 创作规则、文风方向 |
| 系统指令.md | ✅ | 预置，通常不需修改 |

### 2. 逐项引导

对每个缺失或不完整的文件，按以下引导模板提问。

**引导原则**：
- 每次只问 1-2 个问题
- 提供示例回答帮助理解
- 用户回答后立即写入对应文件
- 一个文件完成后自动进入下一个

#### 世界背景.md 引导问题

```
1. 故事发生在什么样的世界？（奇幻/科幻/现代/古代/架空？）
   示例：一个以「灵气」为能源的东方玄幻世界

2. 这个世界的社会结构是什么样的？（有等级制度吗？势力怎么分布？）
   示例：九重天等级体系，天庭统治，四方妖域割据

3. 当前世界处于什么状态？（和平/战争/动荡/变革？）
   示例：天庭内部分裂，妖族趁机崛起，人间界夹缝求生
```

#### 角色.md 引导问题

```
1. 主要角色有哪些？（列出名字和一句话定位）
   示例：萧衍 - 废太子，身怀龙脉之秘

2. 每个角色的核心特征？（年龄/身份/性格/外貌/动机）
```

#### 主角.md 引导问题

```
1. 主角叫什么？多大年纪？
2. 主角当前的处境是什么？
3. 主角有什么特殊之处？（血脉/能力/秘密/使命）
```

### 2.5 方向定调 ⭐

在基本世界观和角色框架就绪后，在填写风格指南之前，先确认以下**创作方向**决策。

**引导原则**：
- 每个方向问 1 个问题，等待用户回答后再进入下一个
- 提供选项和示例帮助理解
- 回答即时写入 `故事/rp-故事名/设定/风格指南.md` 的「创作方向」章节

#### 方向一：基调选择

```
你希望这个故事的情感基调是哪种方向？
（可以组合，选主导方向即可）

① 纯爱   —  温柔细腻，注重情感连接和双向心动
② 后宫   —  多角色攻略，每个角色有独立情感线
③ 黑暗   —  侧重权力博弈和阴谋算计
④ 背德   —  禁忌关系驱动，伦理冲突为核心燃料
⑤ 混合   —  以上几种的组合（请说明）

推荐组合示例：冒险+悬疑 = 探索未知；史诗+黑暗 = 宏大悲剧
```

#### 方向二：主角详细设定

```
主角除了名字和年龄，还有什么突出特质？

身份选项参考：
- 学生（高中生/大学生/研究生）
- 社畜（普通职员/高管/创业者）
- 特权阶层（富二代/官二代/总裁）
- 特殊身份（警察/医生/教师/艺术家等）

身体特质选项参考（可选）：
- 普通体型 / 健硕 / 修长 / 美型 / 其他

性格关键词：
- 温柔 / 霸道 / 腹黑 / 阳光 / 内向 / 闷骚 / 其他
```

#### 方向三：NTR / NTL 选择

```
当涉及他人伴侣时，你希望主角处于什么位置？

① NTL（寝取り Netori）— 主角主动夺取他人伴侣
   → 主角是行动者，攻略别人的女友/妻子/母亲
   
② NTR（寝取られ Netarare）— 主角的伴侣被别人夺走
   → 主角是承受者，体验被背叛/被剥夺的痛苦快感

③ 不涉及 — 不碰已有伴侣的角色
```

#### 方向四：选开内容确认

```
以下选开内容是否启用？

① 近亲DLC — 主角的直系/旁系女性亲属可攻略
   开启后：母亲/姐妹/姑姑/表姐等成为潜在对象
   关闭后：所有有血缘关系的角色自动排除

② 男性催眠体系 — 主角也可以催眠男性
   开启后：男友/丈夫/父亲等男性角色可被催眠
   关闭后：催眠仅对女性有效

③ 状态栏 — 每个回复底部输出角色状态栏
   开启后：每次回复附带格式化的角色状态信息
   关闭后：正常叙事输出
```

#### 风格指南.md 引导问题

```
1. 你希望故事的叙事视角是？（第一人称/第三人称有限/第三人称全知/多视角）
2. 你希望的文字风格是？（华丽/简洁/文艺/口语化/古风/现代）
3. 你希望故事的基调是？（轻松/沉重/悬疑/热血/色气/黑暗）
4. 你希望故事的基调是？（轻松/温暖/紧张/深沉/混合）
5. 你希望剧情侧重？（角色成长/世界观探索/权力斗争/人际关系/混合）
```

### 3. 输出确认

每个文件填充后，告知用户已写入，并简要概括核心内容。

所有文件完成后，提示用户进入下一阶段（文风训练）。

## 输入

无（直接询问用户）

## 输出

`故事/rp-故事名/设定/` 下的 5 个文件
