### COT-基础思维

```
剧情设计:
  - 重构现状: 重构上一幕最末尾的情况，判断用户输入是否超出了{{user}}的能力范畴？
    注意：用户输入只能控制{{user}}决策和行动，无法改变其他人或事物。
    例如："她过来亲我"是不合理的——因为用户不能控制NPC的行动。
    输出格式：上一幕最末尾的情况是xxx，根据用户输入进行能力范畴判定xxx

  - 性格分析: 本次创作涉及的角色（禁止对{{user}}进行分析）的基础信息、性格、深层需求是什么样的？

  - 剧情推理: 结合上述所有梳理内容，从重构现状出发，推断涉及的角色接下来最可能如何思考与行动？
    推理结束后，合理设计并输出时间流动情况。
    例如：时间从x年x月x日x时x分流动到a年a月a日a时a分（禁止时间倒流）

  - 反省: 从上一轮输出中找到违规内容，有几个？犯了什么错误？
    如果错误 > 3：进入正文前先思考如何避免每个错误，思考一次写一句话。
    否则：简单反省即可。

  - 字数检查: 正文字数要求是？列出满足字数的方案。

  - 文风检查: 重新阅读文风所有要求，思索如何执行。

  - OOC检查: 该故事的题材是什么？措辞会不会脱戏？
```

**可提取到 AGENTS.md #7 的内容：**

- "重构现状" → 步骤① 加载上下文
- "性格分析" → 步骤② 角色动机推演
- "剧情推理 + 时间流动" → 步骤③ 输出正文（含时间推进）
- "反省 + OOC检查" → 步骤④ 更新状态前的自检
