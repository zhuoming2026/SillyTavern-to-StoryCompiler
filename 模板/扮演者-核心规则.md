## #3 写作流程

### 每次动笔前：
- 确认当前时间/地点/在场角色
- 确认角色当前状态

### 输出：
- 冒险模式：直接写正文 → 末尾 2-4 个选项
- 导演模式：先规划 → 用户确认 → 写正文
- 选项至少有一个留在当前场景

### 完成后：
- 自动执行 `#更新状态`（保存进度到世界概览/角色状态/事件日志）
- 如发现未更新 → 手动说 **`#更新状态`** 补救
- 说 **`#整理小说`** 将聊天整理到章节

## #引导 初始化向导

用户说 `#引导` 时，逐项检查状态机，引导用户完成故事初始化。

```
□ ① 主角设定
□ ② 文风确认
□ ③ MVU 识别
□ ④ 写作就绪
```

### 逐项引导

**① 主角设定：**
- 检查 `设定/主角.md` 是否已填写
- 如未填写 → "你的角色叫什么？是什么身份？和剧中人什么关系？"
- 填写后 → 更新主角.md → 勾选 ①

**② 文风确认：**
- 展示当前文风概要（来自 `设定/风格指南.md`）
- "文风确认：要调整吗？"
- 确认后 → 勾选 ②

**③ MVU 识别（从角色卡 JSON 提取，不依赖 stat_data.js）：**
- 读取 `设定/角色卡原始数据.json`
- 搜索四个特征：`MagVarUpdate`、`registerMvuSchema`、`[initvar]`、`StatusPlaceHolderImpl`
- 找到任意一个 → "检测到 MVU 变量系统。用 #更新状态 查看/修改变量。"
- 一个都没找到 → "此角色卡没有变量追踪功能。"
- 勾选 ③

**④ 写作就绪：**
- 所有 3 项勾选后
- → "🎭 写作就绪！说 #DM 了解玩法，或直接开始剧情。"

每完成一项，更新 AGENTS.md #11 的勾选状态（`□` → `☑`）。
